# rednote-writing-skill

一个 [Claude Code](https://claude.com/claude-code) skill，把零碎的灵感、想法、素材，整理、延伸、改写成一篇成形的小红书帖子。守你的判断和声音，不替你编内容。

## 这个 skill 做什么

它不是「一键生成爆款」的工具。它只做三件事，把你的碎想顺成结构，把表达提到你自己最顺手的状态，再把没想透的地方延伸下去。判断、立场、真实素材由你提供，措辞和结构交给它重做。它会大胆重写、重组，但绝不编事实。

三种模式：

- **拼贴**：从链接、截图等素材出发
- **共创**：从几句灵感、话术出发，来回迭代成稿
- **审稿**：审一篇你已经写好的草稿

两个特别的设计：

1. **voice 不靠通用模板。** 它用一组真实帖子（`examples.md`）当唯一标准，缝出来的语气、过渡都得像本人，不然不算过。
2. **出稿前过两道关。** 先机械扫违禁标点、AI 味词、平台限流词（避免发出去被悄悄限流没人看到），再做一次「活人感」冷读，专挑读着假、像 AI 写的段落。标题和封面最后才做。

## 文件结构

| 文件 | 装了什么 | 什么时候读 |
| --- | --- | --- |
| `SKILL.md` | 主入口。账号定位、四条铁律（守声音和判断不守原话；大胆延伸但不编事实；写和检查分开；每步朝成稿收口），以及三种模式各自的分步工作流。 | 最先读，每次都读 |
| `examples.md` | 8 篇真实发布过的帖子，每篇标了类型，附怎么开头、怎么转场、怎么把判断和私人感受混着说的注解。voice 的唯一标准。 | 写正文时读 |
| `checks.md` | 硬规则扫描，分四类：违禁标点、AI 味词、结构性 AI 味、小红书违禁词（广告法极限词 + 平台限流词）。机械客观，命中即改。 | 正文写完后才读 |
| `cover-title.md` | 怎么写 2-3 个封面文案和 3-5 个标题，以及死规则（不用破折号、emoji、泛词、爆款词、极限词）。 | 过完两道关后读 |
| `evals.md` | 3 个回归测试场景，以及怎么对着 baseline 跑，确认大改之后 voice 和流程没退化。 | 只给维护者，写帖子时不读 |

## 安装

把这个 repo clone 进你的 Claude Code skills 目录：

```bash
git clone https://github.com/aislinn-yang/rednote-writing-skill.git ~/.claude/skills/xhs-post
```

然后在 Claude Code 里说「帮我写小红书」「我有个灵感」「帮我审稿」之类的话就会触发。

## 改成你自己的 voice

这个 skill 是按一个具体账号（AI 产品增长 x 出海观察）调出来的，`examples.md` 里是该账号的真实帖子。要用成你自己的：

1. 把 `examples.md` 换成你自己的代表作，这是最关键的一步。
2. 改 `SKILL.md` 里的「账号」段落。
3. 按需调 `checks.md` 里的平台限流词库。
4. 「保存到 Notion」是可选的，填你自己的库链接，或整段删掉。
