# 评估场景（作者测试用，不进写帖流程）

> 给作者做回归测试，写帖子时不读此文件。用法见底部。
> 测的是过程：延伸、改写、缝合，以及守住声音和判断。不测"从一句话生成一篇"。
> 每次大改完跑这 3 个场景，对比 baseline，确认 voice 和流程没退化。

## 目录
- 场景1：拼贴（从素材）
- 场景2：共创（从碎想 + 话术）
- 场景3：共创 / 观点向
- 怎么跑 / baseline

## 通用通过标准

- **过程**：先复述 / 对观点对齐，再搭骨架；不一上来甩成稿
- **延伸**：主动深化了想法、指出了想得不够的地方，不只是逐句润色
- **守声音和判断**：成稿保留用户的判断、立场、真实素材，腔调像 examples；措辞和结构可以被大改、提升，不要求保留原句
- **像本人**：对着 examples.md 冷读挑不出明显"非本人"的段落，改写后仍像她、不是书面语 / AI 腔
- **真实细节**：具体名字 / 数据 / 场景来自用户给的料，无编造事实
- **合规**：违禁标点 / AI味词 / 加粗小标题零命中；正文过两道关后才给标题封面；极限词 / 导流 / 黑话零命中（checks.md）

## 场景1：拼贴（从素材出发）

**输入**：一条链接或一段笔记，如"刷到一个 AI 产品在 Reddit 做冷启动的案例 [材料]，帮我拼成一篇"

**期望**：
- 先消化材料，向用户说明"可说的点是 X，角度是不是 Y"，对齐观点再写
- 搭骨架给用户看，缺判断 / 经历就问
- 大胆改写组织，事实来自材料、立场和腔调像用户，参照 examples 干货向（帖子1 / 帖子3）
- 正文 → 两道关 → 标题封面

**失败**：
- 把材料原样翻译 / 摘要，无用户角度（纯搬运无人味）
- 不对观点直接写成《5个Reddit冷启动技巧》清单体
- 编材料里没有的事实；加粗小标题 / "首先其次最后"
- 一上来就给标题封面

## 场景2：共创（从碎想 + 话术出发）

**输入**：零碎想法 + 几句用户话术，如：
> "想写做SEO的人最容易高估外链。话术：'我以前也觉得外链是万能药'。还想说结构比外链重要，但具体怎么展开还没想好，你帮我顺。"

**期望**：
- 先复述脊柱确认："核心是不是，大家高估外链、真正决定排名的是结构？"
- 搭骨架，主动延伸没说透的点（"为什么外链没用""结构怎么复利"），指出哪里想得不够；缺的事实问用户，不编
- 大胆改写和重组用户那几句，提升表达和文学性；判断不变、改写后仍像 examples
- 缝成形 → 两道关 → 标题封面

**失败**：
- 跳过复述和骨架，直接甩成稿
- 改写后变成书面语 / 通用 AI 腔，或把判断 / 立场改没了
- 只逐句润色、不重组不延伸（太保守，不是用户要的）
- 缺的事实自己编
- 写成中立八股或通用 SEO 科普

## 场景3：共创 / 观点向

**输入**：强观点的碎想 + 话术，如：
> "我觉得很多AI产品的增长是伪增长。话术：'我看一个产品增长，第一个问题不是涨了多少，是停了会不会掉'。料：给face swap做SEO三个月7倍、摸清Google靠用户行为校验排名、我自己team全in-house。帮我顺成一篇。"

**期望**：
- 立场鲜明，私人经验和判断混说（参照帖子8），主动延伸论点
- 那句金句可保留也可改得更利落，但判断和力度不减，最好落在收口
- 用给定真实料（7倍、Google机制、in-house），不另编
- 缺的问用户，不脑补

**失败**：
- 中立八股：一段讲伪增长、一段讲"但也有合理性"，无立场
- 金句被改弱 / 淹没在 AI 腔（改写不算错，改弱才算）
- 只润色不延伸
- 套话开头 / 结尾（"在当今AI高速发展的时代""希望对你有帮助"）

## 怎么跑

1. **建 baseline（一次）**：开一个未加载 skill 的干净 Claude，把 3 个输入各跑一次，存下结果作对照。
2. **跑 skill**：开新会话（别用改 skill 的那个），加载 skill，3 个输入各跑一遍。模拟用户互动（它问就答，别给编事实的机会）。
3. **判分**：每篇做一次冷读（最好开干净子 agent，只给成稿 + examples.md，问"像不像本人、哪段最假、判断还在不在、有没有延伸"），逐条对通用标准 + 该场景失败项。
4. **对比**：skill 版应明显比 baseline 更像本人、表达更好、延伸更到位、判断守得住。退化了带着具体哪段回来改，不凭感觉加规则。
